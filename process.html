Testing Methodology

üë•

User Interviews

Participants: 12 communiMap users

Duration: 30-45 minutes each

Focus: Motivation, barriers, and concept reactions

Key Insights:
  • Users want environmental impact to be visible
  • Skepticism about calculation accuracy
  • Preference for community over individual competition
üì±

Prototype Testing

Participants: 8 new users + 6 experienced

Format: Task-based usability testing

Tools: Figma prototypes, screen recording

Key Insights:
  • Navigation needed simplification
  • CO2 calculations required explanation
  • Achievement system resonated well
üìä

Concept Validation

Method: Online survey + focus groups

Participants: 25 active travel advocates

Metrics: Desirability, feasibility, viability

Key Insights:
  • High interest in community challenges
  • Integration approach preferred over standalone app
  • Need for transparent methodology documentation

Methodology Transparency

Implemented

Problem: Users wanted to understand CO2 calculations

Solution: Added detailed methodology page with clear explanations and data sources

Mobile Optimization

Implemented

Problem: Dashboard not optimized for mobile viewing

Solution: Responsive design with mobile-first approach for key features

Guided Goal Setting

Planned

Problem: Users unsure how to set realistic CO2 saving goals

Solution: Onboarding flow with personalized goal recommendations based on historical data

Community Challenges

Implemented

Critical Design Insights

üí°

Biggest Breakthrough

"Make invisible visible" isn't enough

Simply showing CO2 data doesn't create lasting behavior change. You need social context, community connection, and ongoing engagement loops.

‚ö†Ô∏è

Biggest Mistake

Assuming individual motivation

We spent 4 weeks designing personal dashboards before realizing community psychology was far more powerful for sustained behavior change.

üéØ

Biggest Surprise

Trust trumps features

Users cared more about transparent CO2 calculation methods than gamification features. Credibility is foundational for environmental claims.

üîÑ

Process Learning

Theory before features

Starting with behavior change theory would have saved weeks of iteration. Research foundation should come before solution ideation.
